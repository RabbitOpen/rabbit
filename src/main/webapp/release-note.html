ORM 框架 
						
=============================== V1.7.2 =========================================
	
	1、修复ddl功能在web项目中无法扫描到lib包路径的bug。
	2、扩展SpringDaoAdapter对象。
			新增带参数的	createNameMappedQuery
						createNameMappedUpdate
						createNameMappedDelete
			三个方法。
			
			允许传递和SpringDaoAdapter泛型参数不一致的对象作为过滤条件，
			只要字段的名字和类型匹配即可
	3、优化DataSource连接池性能，显著提升连接池访问效率
	
=============================== V1.7.0 ========================================
	
	1、扩展SpringDaoAdapter对象，新增带参数的createQuery、createUpdate、createDelete方法。
	
=============================== V1.6.9.6 =======================================
	
	1、强化了addInnerJoinFilter方法，使之支持OneToMany类型的多端过滤
	
=============================== V1.6.9.5 =======================================
	
	1、替换了orm中所有的StringUtils.isEmpty方法，该方法是spring4中的方法。这样就能支持旧版的spring


=============================== V1.6.9.4 =======================================
	
	1、优化内置别名机制，原有单此查询中关联的表不能超过26个。该版本取消此限制，理论上是任意多表都可以
	2、新增fetch自己的异常提醒
	3、修改DMLFilter showSql方法，将原来的回车符("\r")换成换行符("\n")，解决linux控制台下显示日志不完整的bug。

	1、新增自增长/序列 类型主键字段插入数据库后将字段值set到主键字段的功能

						 mysql		long ----> bigint
						 sqlserver		long ----> bigint
						 db2			long ----> bigint

	3、修复空事务方法抛异常的bug
	4、优化update对象的过滤条件，当单表查询时就不使用子查询

=============================== V1.6.9.2 =======================================
	
	1、修正DDL工具bug。增加db2、sqlserver、oracle对double、float数据类型的支持

RPC 框架

						
=============================== V3.0 ========================================
	1、在3.0-rc版本的基础上调整了rabbit xsd文件定义，将原来比较晦涩的rabbit节点声明变的更通俗易懂
	2、修改HardWorker读取数据的机制。
			
		旧机制：每次读取完ByteBuffer后直接清空，buffer。
			      在极端网络环境下推送channel可能出现截断心跳包的风险。
			      
		新机制: 每次读取ByteBuffer后直接compact。
		
	3、优化服务端线程池机制，调整为：当等待队列中等待任务数超过等待队列长度的10%就新增线程（如果允许的话）来处理

	2、重构spouter推送部分逻辑、在rpc部分开放统一数据回调处理接口RabbitDataHandler，
	        方便其它业务数据的扩展处理
